# Documentation Sync

Synchronize API documentation with the actual OpenAPI specification to detect and report drift.

## Intent
Read the `/openapi.json` endpoint, compare it with `docs/API.md`, and report differences between documented and actual endpoints.

## Inputs
- None (uses the running application)

## Steps

1. **Start application** (if not running):
   ```bash
   # Check if app is running
   curl -s http://localhost:8000/openapi.json > /dev/null 2>&1
   if [ $? -ne 0 ]; then
     echo "Application not running. Starting..."
     make run &
     sleep 3
   fi
   ```

2. **Fetch OpenAPI spec**:
   ```bash
   curl -s http://localhost:8000/openapi.json > /tmp/openapi_latest.json
   ```

3. **Read existing docs**:
   ```bash
   if [ -f "docs/API.md" ]; then
     cat docs/API.md
   else
     echo "# API Documentation\n\n*Generated by docs-sync command*\n"
   fi
   ```

4. **Analyze and compare**:
   - Extract all endpoints from OpenAPI spec
   - Compare with documented endpoints in docs/API.md
   - Identify missing, outdated, or incorrect documentation

5. **Generate diff-like summary**:
   - List endpoints in OpenAPI but not documented
   - List endpoints documented but not in OpenAPI
   - List schema mismatches
   - Generate TODOs for missing documentation

6. **(Optional) Update docs/API.md**:
   - Ask user confirmation before updating
   - Generate comprehensive API.md from OpenAPI spec

## Expected Output

**Example Diff Summary:**
```
Documentation Sync Analysis
============================

Fetching OpenAPI spec from http://localhost:8000/openapi.json...
Reading existing docs/API.md...

Endpoints in OpenAPI but NOT documented:
-----------------------------------------
- GET /notes/search/ - Search notes by query
- PUT /action-items/{id}/complete - Mark action item as complete

Endpoints documented but NOT in OpenAPI:
-----------------------------------------
- POST /notes/{id}/archive - This endpoint no longer exists!

Schema mismatches:
------------------
- POST /notes/:
  - Documented: {title: string, content: string}
  - Actual: {title: string, content: string, tags: string[] (optional)}

Missing documentation for parameters:
-------------------------------------
- GET /notes/search/:
  - Parameter: q (optional, query string) - Not documented!

TODOs for documentation updates:
---------------------------------
1. [ ] Add GET /notes/search/ endpoint documentation
2. [ ] Add PUT /action-items/{id}/complete endpoint documentation
3. [ ] Remove or document POST /notes/{id}/archive
4. [ ] Update POST /notes/ schema to include optional tags parameter
5. [ ] Document the 'q' parameter for search endpoint

Would you like me to update docs/API.md automatically? (y/n)
```

**Example docs/API.md Structure:**
```markdown
# API Documentation

Generated from OpenAPI spec at http://localhost:8000/openapi.json

## Notes Endpoints

### GET /notes/
List all notes.

**Response:**
```json
[
  {
    "id": 1,
    "title": "Welcome",
    "content": "This is a starter note."
  }
]
```

### POST /notes/
Create a new note.

**Request Body:**
```json
{
  "title": "My Note",
  "content": "Note content"
}
```

**Response:** 201 Created with the created note object

### GET /notes/search/
Search notes by query string.

**Query Parameters:**
- `q` (optional): Search query string

**Response:** Array of matching notes

## Action Items Endpoints

### GET /action-items/
List all action items.

### POST /action-items/
Create a new action item.

**Request Body:**
```json
{
  "description": "Do something"
}
```

### PUT /action-items/{id}/complete
Mark an action item as complete.
```

## Safety Notes
- Asks for confirmation before modifying docs/API.md
- Does not modify application code
- Only reads from /openapi.json (read-only endpoint)
- Creates backup of existing docs/API.md before updating

## How to Run
```bash
/docs-sync         # Analyze and show diff
/docs-sync --apply # Also update docs/API.md automatically
```

## Rollback
If docs/API.md was updated, a backup is saved as `docs/API.md.backup`:
```bash
cp docs/API.md.backup docs/API.md
```
